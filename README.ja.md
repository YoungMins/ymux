Windows 向けの軽量な tmux スタイルのターミナルマルチプレクサ。

https://github.com/user-attachments/assets/705fff59-0bda-4460-a87f-d7ba6f50993a

Tauri 2 (Rust) + WebView2 + xterm.js で構築されています。Windows 上で軽量かつ
高速にネイティブ動作しながら、保存されるレイアウト、ペインごとの作業ディレクトリ
と起動コマンド、切り替え可能なシェル (cmd / PowerShell / pwsh / Git Bash / WSL)、
そして各々が独自のレイアウトを記憶する番号付きワークスペースを提供します。

## 機能

- **永続化されるレイアウト**: 再帰的な水平 / 垂直分割。各ペインは自分のシェル、
  `cwd`、任意の起動コマンドを記憶します。
- **カレントディレクトリの継承**: ペインを分割すると、起動時のディレクトリではなく、
  親シェルが現在いるディレクトリで新しいペインが開きます。
  OSC 7 エスケープシーケンスによるリアルタイム追跡を使用しています。
- **シェル自動検出**: システムから `cmd.exe`、Windows PowerShell、
  PowerShell 7 (`pwsh`)、Git Bash、WSL ディストリビューションを検出し、
  選択可能なプロファイルとして提示します。
- **番号付きワークスペース**: `Ctrl+Alt+1` .. `Ctrl+Alt+9` で最初の 9 つの
  ワークスペースを切り替えます。ツールバーの `+` ボタンで上限なくいつでも追加
  でき、ワークスペースタブにマウスを合わせると表示される `×` で削除できます
  (最後の 1 つは削除不可)。すべてのワークスペースは独自のレイアウトを保存します。
  ペインは切り替えを越えて生存し続けるため (tmux スタイル)、REPL や tail が死ぬ
  ことはありません。ワークスペースボタンをダブルクリックすると名前を付けられます。
- **Git worktree ペイン**: コマンドパレットの **「新しい git worktree でペインを
  開く」** でブランチ名を入力すると、git worktree を作成し (デフォルトの場所:
  リポジトリの隣の `.ymux-worktrees/<ブランチ>` ディレクトリ、
  `worktree_base_dir` で変更可能)、その中に移動済みのターミナルペインを開きます
  — AI エージェントや実験ごとに隔離されたチェックアウトを持てます。ペインを
  閉じるとき、またはワークスペース全体を削除するときに worktree の削除を確認
  します (ブランチやコミットには一切触れません)。
- **永続スクロールバック**: ターミナル出力がアプリ再起動後も維持されます。
  各ペインのバッファが (色も含めて) 次回起動時に薄い *「セッション復元」*
  区切り線の下に復元されます。**設定 → 一般** で切り替え可能。
- **エージェント状態を一目で**: 各ターミナルペインが出力の活動とベル / OSC 9
  完了シグナルからプロセスの状態 — 待機 / 実行中 / 完了 / 要注意 — を追跡し、
  色付きのペイン枠線とワークスペースタブのドットで表示します。見ていない
  ペインの CLI が終わると OS 通知と短いビープ音でも知らせます (設定でオフに
  できます)。
- **ペインごとの設定 (⚙)**: 各ターミナルの `⚙` ボタンで**背景色**（カラーピッカー）
  と **HotKey ボタン**（単一行またはバッチコマンド）を管理。背景色は再起動後も
  保持されます。
- **ブラウザペイン**: ツールバーの `+ Browser` ボタンで任意のレイアウトスロットに
  iframe ベースのブラウザを配置できます。戻る / 進む / リロード付きの URL バー。
  URL はワークスペース切替やアプリ再起動を越えて保持されます。
  > **注意:** ブラウザペインは HTML `<iframe>` で実装されているため、
  > `X-Frame-Options` や CSP `frame-ancestors` で埋め込みを拒否するサイト
  > (例: github.com、google.com) は読み込めません。ローカル開発サーバー、
  > Storybook、社内ダッシュボード、API ドキュメント、localhost プレビュー
  > など、**開発用途に適しています**。一般的な Web ブラウジング用ではありません。
- **ペイン拡大**: `Ctrl+Shift+Z` でフォーカス中のペイン以外を隠して集中できます。
  もう一度押すと分割状態に戻ります。
- **スクロールバック検索**: `Ctrl+F` でフォーカス中のターミナルに検索バーを
  開きます。Enter / Shift+Enter で次/前のマッチへ、Esc で閉じます。
- **ペイン名の変更**: `Ctrl+Shift+R` でフォーカス中のペインにカスタムタイトルを
  設定できます。
- **アップデート通知**: バックグラウンドポーラが 6 時間ごとに GitHub リリースを
  確認し、新しいバージョンがあれば閉じられるバナーで通知します。
  自動インストールはしません。
- **システムモニターのステータスバー**: ウィンドウ下部の細いバーが、CPU / RAM /
  GPU / ディスク / ネットワーク ↑↓ を 2 秒ごとにリアルタイム表示します。
  70% 以上で橙、90% 以上で赤にハイライトされます。マルチ GPU / マルチディスクも
  対応 (3 台以下はインライン、それ以上はサマリー + ツールチップ)。
- **Ko-fi 支援ボタン**: `⚙` ボタンの隣にある ☕ Support ボタンから
  [ko-fi.com/youngminkim](https://ko-fi.com/youngminkim) をシステムブラウザで
  開けます。
- **クリック可能な URL**: ターミナル内の `http://` または `https://` リンクを
  `Ctrl+クリック` するとデフォルトブラウザで開きます。
- **設定パネル (⚙)**: WinUI 3 スタイルのモーダルで、左側にメニュー（一般、
  シンタックスカラー、ショートカット、ツール、設定ファイル）、右側にコンテンツ
  が表示されます。表示言語の変更、yCode のシンタックスハイライト用カラー
  パレットをカラーピッカーで編集、内蔵のショートカット/ツールリファレンス
  の確認、`theme.toml` をデフォルトエディタでワンクリックで開けます。
- **コマンドパレット**: `Ctrl+Shift+P` で VS Code スタイルの検索可能な
  コマンドオーバーレイを開きます。名前やキーバインドでファジーマッチ。
- **ワークスペースごとのノート**: 各ワークスペース番号の隣にノートアイコンが
  あり、そのワークスペース専用のノートを開けます。`Ctrl+Alt+N` で現在
  アクティブなワークスペースのノートをトグル。ノートは `localStorage` に
  自動保存されセッションを越えて維持され、内容のあるワークスペースの
  アイコンはアクセントカラーで強調表示されます。
- **クリップボード貼り付け — テキストと画像**: `Ctrl+V` でクリップボード
  テキストをターミナルに貼り付けます。クリップボードに**画像**がある場合
  (例: `Win+Shift+S` のスクリーンショット)、自動クリーンアップされる一時
  ファイルに保存し、引用符で囲んだパスを代わりに入力します — スクリーン
  ショットをペイン内の Claude Code などの AI CLI にそのまま渡せます。
  貼り付けた画像は 24 時間後に削除されます
  (`paste_image_retention_hours` で調整可能)。
- **13言語対応**: English、한국어、日本語、中文、हिन्दी、Español、Français、
  العربية、Português、Русский、Türkçe、Deutsch、Tiếng Việt。ステータスバー
  右下の言語セレクターから変更可能。
- **MSI で PATH 自動登録**: インストールディレクトリがシステム PATH に追加され、
  `ymux`、`ymon`、`ydir`、`ycode`、`ygit`、`y` がすぐ実行可能。
- **軽量**: Tauri バイナリ + WebView2。インストーラのターゲットは 10 MB 未満。

### 同梱 TUI ツール

ymux ターミナルペイン内で実行する独立バイナリ。`y` ランチャーで起動可能
（`y mon`、`y dir`、`y code`、`y git`）。

| ツール | コマンド | 説明 |
|--------|----------|------|
| **ymon** | `ymon` | htop/btop スタイルのシステムモニター（CPU、メモリ、ディスク、プロセス） |
| **ydir** | `ydir` | デュアルペインファイルマネージャー — ナビゲート、コピー/移動/削除、実行ファイル引数ダイアログ |
| **ycode** | `ycode <ファイル>` | TUI コードエディター — シンタックスハイライト（Svelte 含む ~200 言語）、`Ctrl+B` ファイルツリーサイドバー（未保存バッファ確認ダイアログ付き）、undo/redo、検索、goto、Esc終了ダイアログ、CJK/絵文字対応 |
| **ygit** | `ygit` | Git ログ & ブランチビューアー — カラー付きコミットグラフ、ブランチ一覧、チェックアウト |
| **y** | `y help` | インストール済み y* ツールの一覧表示・起動ランチャー |

## 開発

必要なもの: Rust (stable)、Node 20+、pnpm (または npm)。

```sh
pnpm install
pnpm tauri dev          # 開発モードで実行
pnpm tauri build        # Windows MSI インストーラを生成 (Windows 上で実行)
```

Windows 以外のホストでも Rust クレートは `cargo check` がクリーンに通るため、
Linux/macOS でクロスプラットフォームのロジックを開発できますが、完全な
`tauri build` とエンドツーエンドの PTY 検証は Windows 上で行う必要があります。

## 設定

`%APPDATA%\ymux\config.toml` にワークスペース、レイアウト、キャッシュされた
シェルプロファイルが保存されます。構造変更のたびに (デバウンスあり) および
アプリ終了時に書き直されます。

`%APPDATA%\ymux\theme.toml` にはすべての y* ツールが共有するカラーパレット
が保存されます（yMux UI、yCode シンタックスハイライトなど）。**設定 →
シンタックスカラー** のカラーピッカーで編集するか、**設定 → 設定ファイル
→ 開く** でファイルを直接開けます。

## キーボードショートカット

| ショートカット                   | アクション                            |
|----------------------------------|---------------------------------------|
| `Ctrl+Shift+H`                   | ペインを水平に分割                    |
| `Ctrl+Shift+V`                   | ペインを垂直に分割                    |
| `Ctrl+Shift+W`                   | フォーカス中のペインを閉じる          |
| `Ctrl+Shift+Z`                   | フォーカス中のペインを拡大 / 戻す     |
| `Ctrl+Shift+←/→`                 | 前 / 次のペインと位置を入れ替え       |
| `Ctrl+Shift+R`                   | フォーカス中のペインの名前を変更      |
| `Ctrl+Shift+P`                   | コマンドパレットを開く                |
| `Ctrl+Alt+N`                     | アクティブWSのノートを切り替え        |
| `Ctrl+V`                         | クリップボードテキストを貼り付け（画像 → 一時ファイルパス）|
| `Ctrl+F`                         | ターミナルのスクロールバックを検索    |
| `Ctrl+Tab`                       | 次のペインにフォーカス                |
| `Ctrl+Shift+Tab`                 | 前のペインにフォーカス                |
| `Ctrl+Alt+1` .. `Ctrl+Alt+9`     | ワークスペースを切り替え              |
| ワークスペースボタンをダブルクリック | ワークスペース名を変更                |
| URL 上で `Ctrl+クリック`         | リンクをデフォルトブラウザで開く      |
| ツールバーの `⚙` ボタン          | 設定を開く（言語、ショートカット、シンタックスカラー、設定ファイル）|

> **ヒント:** ツールバー右上の `⚙` ボタンを押すと WinUI 3 スタイルの設定
> モーダルが開きます。左サイドバーから表示言語の変更、yCode のシンタックス
> ハイライトパレットの編集、`theme.toml` などの設定ファイルを直接開く操作
> がすべて可能です。
